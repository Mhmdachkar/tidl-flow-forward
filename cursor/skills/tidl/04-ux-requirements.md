# TIDL UX Requirements

## Purpose

This document defines the required user experience for TIDL.

The current implementation focuses heavily on visual design and animation.

The client specifically stated:

> The UI is good, but the UX is wrong.

This means:

- information hierarchy is incorrect
- user flows are missing
- conversion paths are incomplete
- telehealth processes are not visible

The UX requirements below have higher priority than visual design.

---

# Core UX Principle

The user should understand:

1. What TIDL is.
2. What treatment is offered.
3. How it works.
4. Who prescribes it.
5. How they receive it.
6. What happens next.
7. Why they should trust it.

The website is primarily a trust and conversion experience.

---

# User Goals

Users arrive with goals:

- lose weight
- improve health
- improve longevity
- receive legitimate treatment
- avoid complicated injections
- receive medical guidance

The website must answer these goals quickly.

The homepage is not a brochure.

The homepage exists to:

- educate
- build trust
- reduce fear
- explain the process
- drive quiz starts

# User Flow Priority

The following pages are critical:

1. Quiz
2. Product page
3. Checkout
4. Confirmation

The homepage supports these pages.

The homepage is not the final destination.

---

# The Quiz Is The Primary Conversion Tool

The quiz is:

- onboarding
- qualification
- medical intake
- conversion

The quiz is not:

- a marketing quiz
- entertainment
- a lead magnet

Users should understand:

> This information helps a physician review my eligibility.

---

# Quiz UX Requirements

The quiz must:

- feel simple
- reduce anxiety
- show progress
- explain why questions matter

Requirements:

- progress bar
- back button
- save progress
- mobile first
- no dead ends

---

# Product Pages

Product pages answer:

- what is this
- how does it work
- who is it for
- what should I expect
- how is it delivered
- who prescribes it

The goal:

Move the user into the quiz.

---

# Doctor Experience

The user must understand:

- a licensed physician reviews the intake
- prescriptions are real
- approval is not guaranteed

The doctor should feel visible.

The doctor should not feel hidden.

---

# Pharmacy Experience

Users need to understand:

- medicine is prepared by licensed pharmacies
- medications ship safely
- medications arrive properly packaged

The pharmacy process builds trust.

---

# Checkout UX

Checkout must feel:

- secure
- medical
- transparent

Show:

- treatment cost
- shipping
- taxes
- payment methods

Supported:

- cards
- HSA
- FSA

---

# Confirmation Experience

After ordering:

- order confirmation
- physician review status
- expected timelines
- next steps

The customer should never feel uncertain.

---

# Account Portal

Users should see:

- order status
- prescriptions
- appointments
- messages
- reorders

The account should feel simple.

Not like a hospital portal.

---

# Reorder Experience

Reordering should be easy.

The user already has:

- dose
- address
- payment

The website should remove friction.

---

# Trust UX

Medical trust must appear continuously.

Examples:

- physician review
- licensed providers
- secure checkout
- pharmacy partners
- privacy protection

Trust should not be hidden in the footer.

---

# AI Search UX

AI search is:

- product discovery
- education
- recommendations

AI is not:

- diagnosis
- medical advice
- prescriptions

Medical questions escalate to physicians.

---

# Mobile UX

Most users arrive from:

- ads
- social
- search
- AI answers

Mobile users should immediately see:

- headline
- value proposition
- CTA

No excessive spacing.

No dead zones.

---

# Empty Space Rules

Large empty sections create confusion.

Whitespace should:

- support hierarchy
- support focus

Whitespace should never:

- slow the journey
- create uncertainty

---

# CTA Rules

Primary CTA:

Start Assessment

Secondary CTA:

Learn More

Avoid:

- Buy Now
- Add To Cart
- Download App

---

# Trust Before Conversion

The sequence should be:

Trust.

Education.

Process.

Conversion.

Never:

Conversion before trust.

---

# Emotional Journey

The user should move through:

Curiosity
↓
Understanding
↓
Trust
↓
Confidence
↓
Action

---

# Failure Conditions

The UX fails if users think:

- Is this a supplement?
- Is this safe?
- Is this legitimate?
- Who prescribes this?
- Is this approved?
- What happens after payment?

Any unanswered question creates friction.

---

# Final UX Goal

The entire experience should feel:

- simple
- guided
- trustworthy
- medical
- premium

The user should leave the homepage thinking:

> I understand the process.

> I trust the company.

> I know what happens next.

> I feel comfortable starting the assessment.
